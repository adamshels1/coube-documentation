# AI ETA Predictor — Прогноз времени доставки

**Приоритет:** P0
**Компоненты:** Backend (ML-модель), Mobile (отображение ETA), Frontend (отображение ETA)

## Описание

ML-модель, которая предсказывает точное время прибытия на каждую точку маршрута. Динамически обновляется по мере движения водителя. Заказчик видит не просто "В пути", а "Прибудет через 2ч 15мин (точность 92%)".

## Как работает

### Входные данные (features)

**Статические (на момент создания перевозки):**
- Расстояние между точками маршрута (из `route_distance`)
- Тип груза и вес (`cargo_name`, `cargo_weight`)
- Тип кузова (`vehicle_body_type_id`)
- День недели, время суток
- Сезон (зима/лето — влияет на скорость в Казахстане)
- Регион (горная местность, степь, городские маршруты)
- Водитель — средняя скорость по истории (`gis.driver_location`)
- Перевозчик — исторический % выполнения в срок

**Динамические (обновляются в реальном времени):**
- Текущая позиция водителя
- Средняя скорость за последние 30 мин
- Оставшееся расстояние до следующей точки
- Текущее время (учёт ночного/дневного вождения)
- Кол-во оставшихся точек загрузки/разгрузки
- Время на загрузку/разгрузку (из исторических данных)

### Модель

**Подход:** Gradient Boosted Trees (XGBoost/LightGBM)
- Обучение на исторических данных завершённых перевозок
- Target: фактическое время прибытия на точку
- Регулярное переобучение (еженедельно)

**Baseline:** Расстояние / средняя скорость — простая формула для сравнения.

**Advanced:** Учёт паттернов конкретного водителя, сезонности, типа маршрута.

### Расчёт точности (confidence)

```
confidence = 1 - (MAE / mean_actual_time)

Где:
- MAE = средняя абсолютная ошибка модели на валидации
- Рассчитывается per route_type (город, межгород, дальнобой)
```

## Источники данных

| Таблица | Поля | Использование |
|---------|------|---------------|
| `gis.driver_location` | location, timestamp, speed | Текущая позиция, скорость, история движения |
| `applications.cargo_loading` | location, loading_datetime, loading_time_hours | Точки маршрута + фактическое время загрузки |
| `applications.transportation` | cargo_weight, cargo_volume, created_at, status timestamps | Характеристики груза, история статусов |
| `applications.transportation_cost` | tariff_type | Тип тарифа (влияет на маршрут) |
| `gis.transportation_route_history` | route_distance | Расстояние маршрута |
| `gis.cargo_loading_history` | loading_datetime, address | Исторические данные по загрузкам |

## API

### GET /api/v1/ai/eta/{transportationId}

**Response:**
```json
{
  "transportationId": "uuid",
  "predictions": [
    {
      "cargoLoadingId": "uuid",
      "pointType": "LOADING",
      "address": "г. Алматы, ул. Абая 150",
      "plannedArrival": "2026-03-07T14:00:00",
      "predictedArrival": "2026-03-07T14:35:00",
      "confidence": 0.91,
      "delayMinutes": 35,
      "status": "SLIGHT_DELAY"
    },
    {
      "cargoLoadingId": "uuid",
      "pointType": "UNLOADING",
      "address": "г. Астана, ул. Кенесары 40",
      "plannedArrival": "2026-03-08T10:00:00",
      "predictedArrival": "2026-03-08T09:20:00",
      "confidence": 0.85,
      "delayMinutes": -40,
      "status": "ON_TIME"
    }
  ],
  "overallStatus": "SLIGHT_DELAY",
  "updatedAt": "2026-03-07T12:30:00"
}
```

**Статусы:**
- `ON_TIME` — отклонение <10%
- `SLIGHT_DELAY` — отклонение 10-25%
- `SIGNIFICANT_DELAY` — отклонение >25%
- `AHEAD_OF_SCHEDULE` — раньше плана >10%

### GET /api/v1/ai/eta/{transportationId}/history

Возвращает историю изменений ETA (как менялся прогноз по мере движения).

## UI-компоненты

### Mobile (водитель)
- Плашка с ETA до следующей точки на экране заказа
- Цветовая индикация: зелёный (в срок), жёлтый (небольшая задержка), красный (значительная задержка)

### Mobile (заказчик/логист)
- ETA на экране отслеживания водителя (DriverTrackingScreen)
- Push-уведомление при существенном изменении ETA (>15 мин)
- "Поделиться ETA" — ссылка для получателя груза

### Frontend (веб)
- ETA в карточке перевозки
- График изменения ETA на таймлайне перевозки
- Столбец ETA в списке активных перевозок

## Автоуведомления

| Событие | Получатель | Канал | Сообщение |
|---------|-----------|-------|-----------|
| ETA изменился >15 мин | Заказчик | Push | "Время доставки изменилось: новый ETA {время}" |
| ETA изменился >30 мин | Заказчик + логист | Push + SMS | "Значительная задержка: {причина}. Новый ETA: {время}" |
| Водитель прибудет через 30 мин | Контакт в точке разгрузки | SMS | "Водитель прибудет через ~30 мин" |
| Водитель прибудет через 1 час | Получатель груза | Push | "Ваш груз будет доставлен примерно в {время}" |

## Метрики успеха

- Точность ETA (MAPE): <15% для межгорода, <10% для города
- Обновление ETA: каждые 5 мин при движении
- Доступность API: 99.9%
- Улучшение vs. baseline (расстояние/скорость): >30%

## План обучения модели

1. **Сбор данных:** Минимум 500 завершённых перевозок с GPS-треками
2. **Feature engineering:** Извлечение признаков из GPS, маршрутов, грузов
3. **Обучение:** Train/validation/test split 70/15/15
4. **A/B тест:** Показывать ETA для 50% пользователей, сравнить удовлетворённость
5. **Итерация:** Еженедельное переобучение с новыми данными
