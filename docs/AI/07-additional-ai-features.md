# Дополнительные AI-фичи (P2-P3)

## P2 — Средний приоритет

### 1. Мониторинг усталости водителя (Driver Fatigue Detection)

**Описание:** AI-система отслеживает режим труда и отдыха водителя по GPS-данным.

**Алармы:**
- Непрерывное вождение >4 часов без остановки >15 мин
- Суммарное вождение >10 часов за сутки
- Ночное вождение (23:00-06:00) >4 часов
- Менее 8 часов между сменами

**Данные:**
- `gis.driver_location` — timestamp + speed (определение периодов движения/отдыха)
- `gis.employee_latest_loc` — always-on tracking между заказами

**Нормативная база:** Правила режима труда и отдыха водителей РК (Приказ МТК РК).

**Реализация:**
- Анализ скользящего окна GPS-данных
- Определение периодов движения (speed > 5 км/ч) и отдыха
- Push водителю: "Рекомендуем остановиться на отдых"
- Аларм логисту при нарушении нормы

### 2. Автоматическая проверка документов (Document AI)

**Описание:** OCR + AI для автоматической верификации загружаемых документов.

**Документы:**
- Удостоверение личности → извлечение ФИО, ИИН, срок действия
- Водительское удостоверение → категории, срок действия
- Тех. паспорт → марка, гос. номер, VIN
- Доверенность → сроки, полномочия

**Технологии:**
- Google Cloud Vision / Yandex Vision для OCR
- Кастомная модель классификации типа документа
- Rule-based валидация извлечённых полей

**Результат:**
- Автозаполнение полей профиля при загрузке документа
- Валидация: данные в документе совпадают с данными в профиле
- Алерт при истечении срока документа за 30 дней

**Данные:**
- `file.file_meta_info` — загруженные файлы
- `users.employee` — данные профиля для сверки
- `transport.vehicle` — данные транспорта для сверки

### 3. Умные уведомления (Smart Notifications)

**Описание:** AI-оптимизация доставки уведомлений.

**Возможности:**

**Оптимальное время отправки:**
- Анализ `notifications.notification_log` (opened, clicked по времени)
- Модель предсказания вероятности прочтения в зависимости от времени
- Отправка в персональное "окно внимания" каждого пользователя

**Оптимальный канал:**
- Для каждого пользователя определить предпочтительный канал
- Push vs SMS vs Email — по conversion rate из исторических данных
- Каскад: сначала Push, если не прочитал за 15 мин → SMS

**Персонализация контента:**
- A/B тестирование шаблонов уведомлений
- Подбор tone of voice (формальный/неформальный) по engagement

**Данные:**
- `notifications.notification_log` — channel, status, opened, clicked, delivery_attempts
- `notifications.user_delivery_channels` — предпочтения + quiet hours

### 4. Детектор аномалий и мошенничества (Fraud Detection)

**Описание:** AI-система обнаружения подозрительных паттернов.

**Типы мошенничества:**

**Накрутка пробега:**
- Фактический GPS-трек значительно длиннее оптимального маршрута
- Алгоритм: сравнение `route_distance` из GPS vs. Yandex Router
- Порог: отклонение >30%

**Фиктивные перевозки:**
- GPS-трек не соответствует заявленному маршруту
- Слишком быстрое "завершение" перевозки
- Одинаковые GPS-паттерны у разных перевозок одного перевозчика

**Ценовой сговор:**
- Аномально высокие цены от группы перевозчиков на одном направлении
- Статистический анализ распределения цен + обнаружение кластеров

**Двойные SOS:**
- Частые SOS-сигналы от одного водителя/перевозчика
- Анализ паттерна: SOS → отмена → новая перевозка с наценкой

**Реализация:**
- Isolation Forest для обнаружения аномалий в GPS-треках
- Statistical process control для ценовых аномалий
- Rule-based + ML ensemble для комплексной оценки

**Данные:**
- `gis.driver_location` — GPS-треки
- `applications.transportation_cost` — цены
- `applications.transportation` — статусы, сроки

## P3 — Перспективные

### 5. AI-чатбот для поддержки

**Описание:** Интеллектуальный помощник для пользователей платформы.

**Каналы:** Веб (iframe), мобильное приложение, Telegram-бот, WhatsApp.

**Возможности:**
- Ответы на FAQ (статус перевозки, документы, оплата, как создать заявку)
- Помощь с оформлением заявки через диалог
- Проверка статуса перевозки по номеру
- Навигация по платформе ("как посмотреть отчёт?")
- Эскалация на оператора при сложных вопросах

**Технологии:**
- LLM (Claude API / GPT) с RAG на базе знаний платформы
- Интеграция с API Coube для получения данных в реальном времени
- Function calling для выполнения действий (создать заявку, отправить документ)

**Данные для RAG:**
- `coube-documentation/` — база знаний, FAQ
- API-документация — для function calling
- История обращений — для улучшения модели

### 6. Прогноз обслуживания транспорта (Predictive Maintenance)

**Описание:** Предсказание необходимости ТО на основе паттернов эксплуатации.

**Данные:**
- Суммарный пробег (из `gis.driver_location`)
- Характер эксплуатации (городской/междугородный — из GPS)
- Возраст транспорта (`transport.vehicle.issue_year`)
- Частота использования (количество перевозок в месяц)

**Алармы:**
- "Пробег 48 000 км — рекомендуем ТО через 2 000 км"
- "Высокая нагрузка за последний месяц — проверьте тормозную систему"

**Реализация:** Rule-based на первом этапе (пороги пробега), ML на втором (с данными о реальных поломках).

### 7. Голосовой помощник для водителя

**Описание:** Hands-free управление заказами для водителя в движении.

**Команды:**
- "Следующая точка" — информация о следующей загрузке/разгрузке
- "Прибыл" — подтверждение прибытия
- "Выехал" — подтверждение отправления
- "Позвони диспетчеру" — звонок контактному лицу
- "SOS" — активация экстренного сигнала
- "Сколько до точки" — ETA до следующей точки

**Технологии:**
- Speech-to-Text (Google/Yandex Speech Kit — поддержка русского и казахского)
- Intent recognition (простая модель на 10-15 интентов)
- Text-to-Speech для ответов

**Безопасность:** Активация только голосом, минимум взаимодействия с экраном.

### 8. Автоматическая генерация отчётов с инсайтами

**Описание:** AI формирует еженедельные/ежемесячные отчёты с текстовыми инсайтами.

**Пример отчёта:**
```
Еженедельный отчёт ТОО "Транс-Логистик" (01-07 марта 2026)

Ключевые показатели:
- Выполнено 23 перевозки (+15% к прошлой неделе)
- Средняя маржа: 18% (стабильно)
- 100% перевозок в срок

Инсайты AI:
1. Направление Алматы→Шымкент показало рост спроса на 25%.
   Рекомендация: увеличить парк на этом направлении.
2. Водитель Петров А.С. — лучший по эффективности (12 перевозок, 0 задержек).
3. Внимание: прогнозируется снижение спроса на стройматериалы через 2 недели.
```

**Технологии:** LLM для генерации текста на основе структурированных данных из отчётов.
