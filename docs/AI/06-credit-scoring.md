# Credit AI — Скоринг для факторинга

**Приоритет:** P1
**Компоненты:** Backend (ML-модель), Frontend (отображение скора), Admin (управление)

## Описание

AI-система оценки кредитоспособности перевозчиков для факторинговых операций. Автоматически рассчитывает риск неплатежа и рекомендуемый лимит финансирования.

## Текущее состояние

Платформа уже имеет богатую базу для скоринга:
- **14-мерный KYC-скоринг** (`users.kyc_score`) — налоги, суды, кредит, ДТП, штрафы и др.
- **Проверка компании** (`users.company_check_data`) — налоговая задолженность, судебные дела, кредитная история, лицензии
- **Проверка водителя** (`users.person_check_data`) — лишение прав, ДТП, судимость, медицинские ограничения
- **История платежей** — данные по оплатам и задержкам
- **Факторинговые операции** (`factoring.*`) — история пользования факторингом

**Проблема:** Решение о факторинге принимается вручную. AI может автоматизировать и улучшить точность.

## Модель

### Входные данные (features)

**KYC-блок (из `users.kyc_score`):**
- legal_status_score — юридический статус
- tax_discipline_score — налоговая дисциплина
- lawsuits_score — судебные дела
- credit_history_score — кредитная история
- factoring_score — история факторинга
- dtp_fines_score — ДТП и штрафы
- customer_reviews_score — отзывы заказчиков
- order_history_score — история заказов
- documents_score — полнота документов

**Финансовый блок:**
- Средний объём перевозок за 3/6/12 месяцев
- Средняя стоимость перевозки
- Средняя задержка оплаты (payment_delay из `transportation_cost`)
- Процент перевозок с задержкой оплаты
- Текущая дебиторская задолженность
- История использования факторинга (суммы, возвраты)

**Операционный блок:**
- Количество завершённых перевозок
- % перевозок в срок
- % отменённых перевозок
- Количество SOS-инцидентов
- Давность регистрации на платформе

### Целевая переменная

- **Binary:** Был ли дефолт (задержка >60 дней) по факторинговой операции
- **Regression:** Количество дней задержки оплаты

### Алгоритм

**Модель:** Gradient Boosting (XGBoost)
- Устойчив к пропускам в данных
- Хорошо работает с табличными данными
- Интерпретируемость через SHAP values

**Выход:**
```
credit_score: 0-100  (0 = высокий риск, 100 = низкий риск)
risk_category: LOW / MEDIUM / HIGH / CRITICAL
recommended_limit: сумма в тенге
confidence: 0.0-1.0
```

### Расчёт лимита

```
base_limit = avg_monthly_revenue * 0.7
risk_multiplier = credit_score / 100
recommended_limit = base_limit * risk_multiplier
max_limit = cap по политике компании
final_limit = min(recommended_limit, max_limit)
```

## UI-компоненты

### Frontend — Профиль перевозчика
- Бейдж кредитного скора (цвет по категории риска)
- Список ключевых факторов (положительные/отрицательные)
- График изменения скора за время

### Frontend — Факторинг
- Автоматическое отображение рекомендуемого лимита
- "Одобрить" / "На проверку" / "Отклонить" на основе скора
- Объяснение решения AI

### Admin — Управление скорингом
- Список всех организаций с кредитными скорами
- Фильтрация по категории риска
- Ручная корректировка лимитов (с логированием)
- Настройка пороговых значений для автоматического одобрения

## Автоматизация решений

| Скор | Категория | Действие |
|------|-----------|----------|
| 80-100 | LOW | Автоматическое одобрение факторинга |
| 60-79 | MEDIUM | Одобрение с ручной проверкой |
| 40-59 | HIGH | Ручная проверка обязательна |
| 0-39 | CRITICAL | Автоматический отказ |

## Метрики успеха

- Точность предсказания дефолта: AUC-ROC > 0.85
- Снижение дефолтов по факторингу: -40%
- Время принятия решения по факторингу: с 2 дней до 5 минут
- False positive rate (отказ хорошим клиентам): <5%
