# Задача 10: Hardcoded пароль в Keycloak -> генерация

**Приоритет:** Critical (Password Management — 1 проблема)
**Риск поломки:** Высокий
**Компонент:** coube-backend

## Проблема

В `KeycloakUserService.java:34` при создании пользователя используется захардкоженный пароль `"password"` для всех пользователей.

> **Контекст:** Система использует OTP (SMS) для аутентификации, поэтому Keycloak-пароль фактически не используется пользователями напрямую. Но его наличие — уязвимость.

## Затронутый файл

### `coube-backend/src/main/java/kz/coube/backend/auth/keycloak/service/KeycloakUserService.java`

**Строка 34:**
```java
// БЫЛО:
.credentials(List.of(Credential.fromPassword("password"))) // TODO: mock otp password
```

**СТАЛО:**
```java
.credentials(List.of(Credential.fromPassword(java.util.UUID.randomUUID().toString())))
```

## Пояснение

- `UUID.randomUUID()` генерирует уникальный случайный пароль для каждого нового пользователя
- Пароль невозможно предсказать или подобрать перебором
- Пользователь аутентифицируется через OTP, поэтому этот пароль никогда не вводится вручную
- Если в будущем потребуется прямой вход по паролю — реализовать отдельный flow смены пароля

## Важно

Это изменение влияет **ТОЛЬКО на создание новых пользователей**. Существующие пользователи с паролем `"password"` останутся как есть.

Для существующих пользователей (опционально, в будущем):

