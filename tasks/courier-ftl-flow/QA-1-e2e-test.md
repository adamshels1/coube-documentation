# QA-1 — End-to-end тест полного курьерского FTL-флоу

**Статус:** 🔴 Критично  
**Оценка:** 1 день  
**Зависит от:** Все BE и FE задачи  
**Окружение:** Stage

---

## Описание

Полный сценарий тестирования нового курьерского FTL-флоу на stage окружении.

**Тестовые аккаунты:**
- Заказчик: `+77470709343`
- Перевозчик: `+77478777626`
- Курьер: сотрудник организации перевозчика с ролью DRIVER

---

## Чеклист

### Шаг 1 — Заказчик создаёт заявку

- [ ] Логин под заказчиком `+77470709343`
- [ ] Переход в раздел создания заявки
- [ ] Выбор типа `COURIER_DELIVERY`
- [ ] Добавление 2-3 точек маршрута (адреса в Алматы)
  - Точка 1: LOADING (склад), адрес, контакт
  - Точка 2: UNLOADING (адрес получателя), контакт, требуется фото
  - Точка 3: UNLOADING (второй получатель)
- [ ] Заполнение стоимости маршрута (например 5000 тенге)
- [ ] Опционально: включить факторинг
- [ ] Публикация заявки
- [ ] ✅ Заявка появляется в списке со статусом `CREATED`

### Шаг 2 — Перевозчик видит и откликается

- [ ] Логин под перевозчиком `+77478777626`
- [ ] Переход в раздел заявок
- [ ] Фильтр по типу `COURIER_DELIVERY`
- [ ] ✅ Заявка заказчика видна в списке
- [ ] Открыть детали заявки, просмотреть маршрут
- [ ] Отправить встречное предложение (например 4500 тенге)
- [ ] ✅ У заказчика заявка переходит в `WAITING_CUSTOMER_DECISION`

### Шаг 3 — Заказчик принимает и подписывает

- [ ] Логин под заказчиком
- [ ] Открыть заявку, просмотреть отклик перевозчика
- [ ] Принять предложение
- [ ] ✅ Создаётся Contract
- [ ] Подписать договор через EDS (QR или NCALayer)
- [ ] ✅ Статус обновляется, перевозчику отправляется на подпись

### Шаг 4 — Перевозчик подписывает договор

- [ ] Логин под перевозчиком
- [ ] Подписать договор через EDS
- [ ] ✅ Статус `SIGNED_CUSTOMER`
- [ ] ✅ Появляется кнопка "Назначить курьера"

### Шаг 5 — Перевозчик назначает курьера

- [ ] Нажать "Назначить курьера"
- [ ] Выбрать курьера из списка сотрудников
- [ ] ✅ Статус `WAITING_DRIVER_CONFIRMATION`
- [ ] ✅ Курьер получает push-уведомление / SMS

### Шаг 6 — Курьер принимает на мобилке

- [ ] Логин под курьером в мобильном приложении
- [ ] ✅ Заявка видна в списке активных заказов
- [ ] Принять заказ
- [ ] ✅ Статус `DRIVER_ACCEPTED`

### Шаг 7 — Курьер исполняет маршрут

- [ ] Начать маршрут → статус `ON_THE_WAY`
- [ ] Прибытие на точку 1 (LOADING)
- [ ] Отбытие с точки 1
- [ ] Прибытие на точку 2 (UNLOADING)
- [ ] Если `isPhotoRequired=true` — загрузить фото
- [ ] Пометить заказ как `DELIVERED`
- [ ] Отбытие с точки 2
- [ ] Прибытие на точку 3, аналогично
- [ ] ✅ Все заказы `DELIVERED`

### Шаг 8 — Завершение

- [ ] ✅ Маршрут переходит в статус `FINISHED`
- [ ] ✅ Создаётся Invoice
- [ ] ✅ Создаётся Act
- [ ] Если факторинг включён → ✅ FactoringPayoutRequest создаётся

### Шаг 9 — Проверка данных в БД

```bash
# Проверить статус транспортировки
PGPASSWORD='platform.c0ube.kz' psql -h localhost -p 15432 -U coube-usr -d coube-db -c \
  "SELECT id, number, status, transportation_type, source_system, executor_employee_id 
   FROM applications.transportation 
   WHERE transportation_type = 'COURIER_DELIVERY' AND source_system IS NULL 
   ORDER BY created_at DESC LIMIT 5;"
```

- [ ] `status = 'FINISHED'`
- [ ] `transportation_type = 'COURIER_DELIVERY'`
- [ ] `source_system IS NULL` (ручное создание)
- [ ] `executor_employee_id IS NOT NULL` (курьер назначен)
