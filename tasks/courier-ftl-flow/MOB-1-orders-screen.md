# MOB-1 — Проверить MyOrdersScreen: заказы нового флоу

**Статус:** 🟡 Важно  
**Оценка:** 1ч  
**Файл:** `coube-mobile/src/screens/MyOrdersScreen/MyOrdersScreen.tsx`

---

## Описание

Убедиться что заказы `COURIER_DELIVERY` из нового FTL-флоу (где `source_system=null`) попадают в список курьера наравне с заказами из импорта.

---

## Чеклист

### Анализ текущего кода

- [ ] Прочитать `MyOrdersScreen.tsx` — как формируется список активных заказов
- [ ] Найти API вызов для получения списка заказов водителя
- [ ] Проверить есть ли фильтрация по `sourceSystem` или `externalWaybillId`

### Проверки

- [ ] Нет жёсткой фильтрации `sourceSystem != null` в списке
- [ ] Нет жёсткой фильтрации `externalWaybillId != null`
- [ ] Карточка заказа (`OrderCard`) корректно рендерится без `externalWaybillId`
- [ ] Номер заявки из системы Coube (внутренний `transportation.number`) показывается как идентификатор

### Если нужны правки

- [ ] Сделать `externalWaybillId` опциональным в типах (`CourierOrder`, `OrderDetails`)
- [ ] Если `externalWaybillId === null` → показывать внутренний номер заявки
- [ ] Убедиться что нет краша при рендере карточки без трек-номеров

### Проверка

- [ ] Заказ из нового FTL-флоу видно в списке курьера на мобилке
- [ ] Нет runtime ошибок в консоли при открытии экрана
