# FE-5 — Страница деталей: цепочка оффер → договор → курьер

**Статус:** 🔴 Критично  
**Оценка:** 4ч  
**Зависит от:** FE-1, FE-2, BE-3  
**Файлы:**
- `coube-frontend/src/views/Applications/` (страница деталей заявки)
- `coube-frontend/src/components/ContractSigningFlow/ContractSigningFlow.vue`

---

## Описание

На странице деталей курьерской заявки обеспечить полную цепочку взаимодействия для заказчика и перевозчика.

---

## Чеклист

### Страница деталей — для заказчика (статус CREATED)

- [ ] Показывать список откликов перевозчиков (`GET /api/v1/customer/transportation/{id}/transportation-costs`)
- [ ] Карточка отклика: организация перевозчика, предложенная цена, условия
- [ ] Кнопка "Принять предложение" → `POST /api/v1/customer/accept-price/{costId}`
- [ ] После принятия → создаётся Contract, статус → `WAITING_CUSTOMER_DECISION`

### Подписание договора (статус WAITING_EXECUTOR_SIGN / WAITING_CUSTOMER_SIGN)

- [ ] Отображать блок подписания с `ContractSigningFlow.vue`
- [ ] Возможность скачать/просмотреть договор (PDF) перед подписью
- [ ] После подписи обеими сторонами → статус `SIGNED_CUSTOMER`

### Страница деталей — для заказчика (статус SIGNED_CUSTOMER)

- [ ] Показывать информацию о перевозчике
- [ ] Показывать статус назначения курьера (ожидается от перевозчика)
- [ ] После назначения курьера — показывать имя и телефон курьера

### Страница деталей — для перевозчика (статус CREATED)

- [ ] Перевозчик видит детали заявки: маршрут, точки, требования
- [ ] Форма отклика: ввод цены (встречное предложение)
- [ ] Кнопка "Принять цену заказчика" → `POST /api/v1/executor/accept-price/{id}`
- [ ] Кнопка "Отправить встречное предложение" → `POST /api/v1/executor/counter-offer/{id}`

### Страница деталей — для перевозчика (статус SIGNED_CUSTOMER)

- [ ] Показывать кнопку "Назначить курьера" — открывает `CourierAssign` модал
- [ ] Модал вызывает `POST /api/v1/executor/{id}/assign-courier` (не `/courier/waybills/{id}/assign`!)
- [ ] После назначения — показывать статус `WAITING_DRIVER_CONFIRMATION`

### Отображение точек маршрута

- [ ] Список точек: адрес, тип (погрузка/разгрузка), время, контакт
- [ ] Карта с маршрутом (опционально, если есть координаты)

### Проверка

- [ ] Полная цепочка кликабельна в браузере без ошибок в консоли
- [ ] ContractSigningFlow корректно отрабатывает для COURIER_DELIVERY
