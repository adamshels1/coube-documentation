## 2. ЧТО ТРЕБУЕТ WL KTS vs ЧТО ЕСТЬ

### Виды перевозок Keden

```
1. Гружёный контейнер: ТЕРМИНАЛ → СКЛАД КЛИЕНТА
   (включает 2 этапа: доставка гружёного + возврат порожнего)

2. Гружёный контейнер: СКЛАД → ТЕРМИНАЛ
   (забор контейнера и доставка для ЖД-отправки)

3. Порожние контейнеры: ТЕРМИНАЛ ↔ ТЕРМИНАЛ
   (городские потоковые перевозки для формирования поездов)

4. Междугородние перевозки фурами (тенты)
   (FTL — уже поддерживается COUBE)

5. Мультимодальные цепочки: АВТО + ЖД
   (новый тип, требует интеграции с ЖД-операциями)
```

## 3. ГРУППИРОВКА ДОРАБОТОК ПО БЛОКАМ

### БЛОК А: White Label инфраструктура
- Создать SCSS-тему Keden (цвета, логотип, favicon)
- Настроить домен platform.keden.kz
- Развернуть отдельный Keycloak realm `keden-trans-service`
- Развернуть отдельную PostgreSQL-базу (полная изоляция данных)
- Настроить отдельный S3-бакет для файлов Keden
- Развернуть отдельные Docker-контейнеры (бэк + фронт + мобилка)
- Настроить CI/CD для отдельной сборки

### БЛОК Б: Новые роли и доступы
- Расширить `KeycloakRole` enum: DISPATCHER, TERMINAL_OPERATOR, WAREHOUSE_OPERATOR
- Создать маппинг ролей в Keycloak realm Keden
- Настроить AccessControl для новых ролей (что видит и может каждая роль)
- Интерфейсы бэкенда/фронтенда/мобилки с учётом новых ролей:
  - **Диспетчер Keden** — управление рейсами, назначение машин, мониторинг
  - **Оператор терминала** — слоты, очередь, разрешения, погрузка
  - **Оператор склада клиента** — приёмка, выгрузка, подтверждение сдачи
  - **Корпоративный клиент** — отслеживание контейнеров, ETA, статус разрешений

### БЛОК В: Контейнер и разрешения (SFM ядро)
- Сущность «Контейнер» (полный номер, владелец, тип, статус, история)
- Обязательная сверка номера на всех этапах
- QR-код / RFID / камера для сканирования в мобильном приложении
- Система разрешений Clearance (СВХ, КТЖ, КДТС, ТХК)
- Цифровые подтверждения от каждой стороны
- Блокировки: нет разрешений → нет машины → нет заезда → нет выезда

### БЛОК Г: Слоты, очередь и погрузка
- Временные слоты для заезда на терминал
- Электронная очередь с приоритетами (Keden vs сторонние)
- Автоперераспределение при опоздании
- Назначение: машина → контейнер → слот
- ETA прибытия машины для крановщика
- Подготовка контейнера заранее по ETA
- Фиксация начала/окончания погрузки
- Очередь в мобильном приложении водителя

### БЛОК Д: Контроль рейса, выгрузка, порожний
- Этапы контейнерного рейса: в пути → склад → выгрузка → выехал → сдача порожнего → завершён
- Прогноз ETA с учётом пробок и отклонений
- Видимость для клиента: движение + ETA + подготовка склада
- Проверка пломб, фото контейнера, замер выгрузки
- Комиссионная выдача как отдельный статус
- Сдача порожнего: адрес в системе, навигация, фото/цифровое подтверждение
- Рейс завершён только после: сдача груза + сдача порожнего

### БЛОК Е: Водители, инциденты, допуск
- Мед. допуск водителя (подтверждение медработником в системе)
- Блокировка назначения без допуска
- Структурированные инциденты (8+ типов) с фото и привязкой к рейсу
- Фиксация нарушений: скорость, маршрут, простои
- Разделение ответственности (водитель vs склад vs терминал)
- История работы и инцидентов по водителю

### БЛОК Ж: Топливо, пробег, техконтроль
- Пробег только по GPS (исключение ручного ввода)
- Нормативы топлива: маршрут + расстояние + тип ТС
- Привязка заправок к рейсу с лимитом
- Контроль сливов топлива (датчики уровня, если есть)
- Фиксация техсобытий (вскрытие баков, отключение датчиков)
- Аналитика аномалий: нетипичные маршруты, перерасход, подозрительные остановки

### БЛОК З: Интеграции Keden
- Адаптация интеграции 1С под бухгалтерию Keden (выгрузка рейсов, финансов)
- Интеграция ЕСУДТ-ТТН (электронные товаро-транспортные накладные)
- IoT/ГЛОНАСС: приём данных от датчиков топлива и CAN-шины
- Интеграция с терминалами (API слотов, разрешений)
- Мультимодальные цепочки: связка авто-рейса с ЖД-отправкой

### БЛОК И: Аналитика, KPI и мотивация
- Полная модель рейса: время, простои, клиент, машина, водитель, контейнер
- Финансовая модель: доход, себестоимость, топливо, маржа, экономика на 1 контейнер / 1 км
- Отчёты по типам перевозок: гружёные / порожние / межгород / мультимодальные
- KPI: среднее время рейса, SLA, инциденты, расход топлива, утилизация
- Автоматический расчёт ЗП, премий, бонусов по данным системы
- Прозрачность по ролям: водитель / диспетчер / менеджер / клиент

| Задача | Что делает ИИ | Приоритет |
|--------|--------------|-----------|
| **Прогноз ETA** | ML-модель предсказывает время прибытия машины на терминал/склад (история, пробки, время суток) | Высокий |
| **OCR номера контейнера** | CV-модель автоматически считывает номер контейнера по фото/видео камеры водителя | Высокий |
| **Оптимизация очереди терминала** | Автоматическое распределение слотов с минимизацией простоев | Высокий |
| **Обнаружение аномалий** | Выявление необычных маршрутов, подозрительных остановок, аномального расхода топлива | Средний |
| **Прогноз загрузки терминала** | Предсказание потока контейнеров для планирования ресурсов (краны, персонал) | Средний |
| **ИИ-ассистент диспетчера** | Подсказки по оптимальному назначению машин, реакция на инциденты | Средний |
| **Анализ причин задержек** | Кластеризация: конкретный терминал, склад, время суток, водитель | Средний |
| **Предиктивное ТО** | Прогноз поломок по данным IoT/ГЛОНАСС и истории | Низкий (след. этап) |

### Рекомендуемые технологии

| Задача | Технология | Примечание |
|--------|-----------|------------|
| Прогноз ETA | XGBoost / LightGBM | Табличные данные + GPS-фичи |
| OCR контейнеров | YOLO v8 + PaddleOCR | Container number recognition |
| Оптимизация очередей | Google OR-Tools / CP-SAT | Scheduling optimization |
| Аномалии | Isolation Forest / DBSCAN | GPS-треки, расход топлива |
| ИИ-ассистент | Claude API | Русский язык, анализ ситуации |
| Прогноз загрузки | Prophet / ARIMA + XGBoost | Временные ряды + сезонность |

### Сроки по ИИ

| Фаза | Срок | Ресурсы |
|------|------|---------|
| Фаза 1: сбор данных | Параллельно (0 доп. нед.) | Архитектор данных |
| Фаза 2: ETA + OCR + аномалии | 8–10 недель | 1 ML-инженер + 1 CV-инженер |
| Фаза 3: оптимизация + ассистент | 8–12 недель | 1 ML-инженер + 1 бэкенд |

- Управление водителями, ТС, сотрудниками
- Система ролей через Keycloak (базовые роли)
- Генерация документов PDF/Excel
- Электронная подпись (NCALayer, eGov Mobile)
- 26 типов отчётов с ECharts + Excel-экспорт
- Фото-фиксация и работа с файлами (MinIO)
- История событий рейса
- Docker-контейнеризация
- Мультитенантная архитектура (Organization + JWT)
- SCSS-тематизация (каркас для Keden-темы)
- Существующая интеграция 1С (Kompra — адаптировать под Keden)

### Что нужно разработать с нуля:
- Сущность «Контейнер» с профилем и сверкой
- Модуль разрешений (Clearance) от нескольких сторон

- QR/RFID/OCR сканирование контейнеров
- Медицинский допуск водителя
- Структурированные инциденты с категориями
- Контроль топлива и GPS-пробега
- 3 новые роли (Dispatcher, Terminal Operator, Warehouse Operator)
- Интеграция IoT/ГЛОНАСС и CAN
- Интеграция ЕСУДТ-ТТН
- Мультимодальные цепочки (авто + ЖД)
- Новые типы отчётов: по контейнерам, терминалам, топливу, KPI водителей
- ИИ-модели (отдельный этап)
