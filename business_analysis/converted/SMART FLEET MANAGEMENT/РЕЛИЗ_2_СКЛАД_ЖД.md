## ЦЕЛЬ РЕЛИЗА

Расширить платформу на уровень **глубокой логистической оркестрации**:
- Складские операции
- Терминальная логистика
- Контейнерные депо
- ЖД-перевозки
- Мультимодальные цепочки (авто + ЖД)

**Результат:** Авто + Склад + Терминал + ЖД → единая система.

White Label COUBE превращается в полноценную цифровую платформу уровня международных терминальных операторов.

## 2. МОДУЛИ РЕЛИЗА 2: ДЕТАЛЬНОЕ ОПИСАНИЕ

### Модуль 1: Warehouse Management Lite — складской модуль

**Готовность: 10% — разработка с нуля**

Облегчённая система управления складскими операциями для контейнерной логистики.

**Функционал:**

**1. Приём контейнера на склад:**
- Регистрация прибытия машины и контейнера
- Проверка пломб — фото/видео фиксация состояния
- Фиксация состояния контейнера при приёме
- Сверка номера контейнера с данными системы (интеграция с Container ID из Р1)

**2. Процессы разгрузки:**
- Назначение ворот для разгрузки
- Назначение склада/бригады
- Фиксация начала и окончания разгрузки
- Время простоя склада — автоматический учёт

**3. Комиссионная выдача:**
- Фиксация повреждений контейнера
- Фиксация вскрытия
- Фото/видео подтверждение
- Отдельный статус в системе

**4. Интеграция с WMS клиента:**
- API для связки со складской WMS клиента
- Двусторонний обмен статусами

**5. Синхронизация с Р1:**
- Связка со статусами рейса из Релиза 1
- Обновление статуса рейса при событиях на складе

---

### Модуль 2: Складские слоты

**Готовность: 0% — разработка с нуля**

**Функционал:**
1. **Временные окна на разгрузку** — назначение слота для каждой машины
2. **Пересчёт очереди** при задержках — автоматическое перераспределение
3. **SLA склада** — контроль времени обработки
4. **Интеграция с ETA** автомобиля из Р1 — склад знает, когда приедет машина
5. **Уведомления** — склад получает оповещение о приближении машины

---

### Модуль 3: Терминальная логистика (Depot / Terminal)

**Готовность: 5% — разработка с нуля**

Дигитализация процессов приёмосдатчиков и терминальных операций.

**Функционал:**
1. **Дигитализация приёмосдатчиков** — цифровое подтверждение приёма/сдачи контейнеров
2. **Просмотр очереди на терминале** — оператор видит все назначенные машины и контейнеры
3. **Подтверждение погрузки/выгрузки** — оператор терминала подтверждает операции в системе
4. **Назначение мест хранения контейнеров** — контейнер привязан к конкретному месту на терминале
5. **Отображение потока порожних контейнеров** — визуализация движения порожних по терминалам
6. **Управление ресурсами терминала** — краны, персонал, площадки

---

### Модуль 4: ЖД-перевозки (Rail Freight Module)

**Готовность: 0% — разработка с нуля**

Полноценный модуль для учёта и управления ЖД-этапами перевозки контейнеров.

**Функционал:**

**1. Регистрация ЖД-контейнера:**
- Номер контейнера (связка с Container ID из Р1)
- Тип контейнера
- Вес
- Маршрут (станция отправления → станция прибытия)

**2. Интеграция с ЖД-операторами:**
- Получение данных о прибытии контейнера
- Статус «готов к вывозу»
- Статус «погружен на платформу»
- API-интеграция с системами ЖД-операторов

**3. Управление мультимодальным циклом:**
- ЖД → авто → склад
- Склад → авто → ЖД
- Автоматическое создание авто-рейса при прибытии контейнера по ЖД

**4. Связь с TMS из Р1:**
- ЖД-этап привязан к авто-рейсу
- Единая история контейнера: ЖД + авто + склад
- Единый статус перевозки

**5. Учёт времени на станциях:**
- Время выгрузки с ЖД-платформы
- Время перегрузки
- Время задержек на станциях
- Причины задержек

---

### Модуль 5: Supply Chain Orchestrator — цепочки поставок

**Готовность: 0% — разработка с нуля**

Модуль оркестрации мультимодальных цепочек поставок.

**Функционал:**
1. **Построение цепочки:** ЖД → авто → склад → авто → терминал
2. **Визуальная карта прохождения контейнера** — где контейнер сейчас и что с ним было
3. **Единый номер цепочки** — все этапы связаны одним идентификатором
4. **Единый статус** — агрегированный статус всей цепочки
5. **Кросс-инциденты** — связь инцидентов всех видов (ЖД, авто, склад) в рамках одной цепочки
6. **Timeline** — хронология всех событий цепочки
7. **Уведомления** — автоматические оповещения при смене этапа

---

### Модуль 6: Финансовый модуль (склад + ЖД)

**Готовность: 25%**

Расширение существующего финансового модуля Coube (счета, акты, реестры, факторинг) на складские и ЖД-операции.

**Функционал:**
1. **Тарификация терминальных операций** — погрузка, выгрузка, хранение, перемещение
2. **Тарификация складских услуг** — разгрузка, хранение, комиссионная выдача
3. **Тарификация ЖД-этапов** — перевозка, погрузка/выгрузка на станции
4. **Маржинальность полной цепочки поставки** — доход - все затраты по всем этапам
5. **Экономика на 1 контейнер** — полная себестоимость мультимодальной перевозки
6. **Автоматическое формирование документов** — счета и акты по складским и ЖД-услугам

---

### Модуль 7: Расширенная аналитика

**Готовность: 30%**

Расширение системы отчётов Coube (26 типов, ECharts, Excel-экспорт) на мультимодальные операции.

**Отчёты:**
1. **Цикл контейнера** — от «заехал» до «сдан порожним» (полный путь)
2. **Статистика складов** — время обработки, простои, SLA, загрузка
3. **Сравнение ЖД-станций** — эффективность, задержки, объёмы
4. **Накопительный оборот контейнеров** — сколько контейнеров обработано за период
5. **Мультимодальные KPI:**
   - Среднее время полной цепочки
   - Доля задержек по этапам (ЖД / авто / склад / терминал)
   - SLA по типам перевозок
6. **SLA складов / терминалов / ЖД** — выполнение нормативов по каждому участнику
7. **Финансовая аналитика** — маржинальность по цепочкам, типам перевозок, клиентам
