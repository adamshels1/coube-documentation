## 2. ЧТО ТРЕБУЕТ MONOTRANS vs ЧТО ЕСТЬ

### FLOW: Полный цикл рейса MONOTRANS

```
ЗАВОД (погрузка, норматив 2ч)
    │
    ├─── [АЗС КазахОйл] ─── ETA/ETD
    │
    ├─── [Точка отдыха] ─── ETA/ETD
    │
    ├─── [ГРАНИЦА] ─── ETA/ETD (только экспорт: КГ/УЗ/ТД)
    │
    ├─── [КЛИЕНТ] ─── слив (норматив 2ч)
    │
    ├─── ОБРАТНЫЙ МАРШРУТ с контрольными точками
    │
    └─── ЗАВОД (готовность к следующей погрузке)
```

### Сводная таблица: что есть / что нужно доработать

| # | Требование MONOTRANS | Текущий статус | Что нужно |
|---|---------------------|---------------|-----------|
| 1 | **Модель рейса** (экспорт + внутри РК, цикличность) | Частично: есть перевозки, маршруты, точки загрузки | Доработать: добавить типы "экспорт"/"внутри РК", цикличность рейсов, привязку к заводу, понятие "обратный маршрут" |
| 2 | **План-график с нормативами ETA/ETD** на каждую контрольную точку | Нет: маршруты есть, но без нормативов и план-факта по времени | **Новый модуль:** справочник нормативов, автоматический расчет ETA/ETD, нормативы погрузки/слива/границы |
| 3 | **Автоперерасчет графика** при отклонениях | Нет | **Новый модуль:** движок пересчета ETA при фактическом отставании |
| 4 | **"Красные участки"** — подсветка просроченных этапов | Нет | Доработка фронта и бэка: логика сравнения факт/план, визуализация |
| 5 | **Контрольные точки маршрута** (АЗС, граница, отдых, клиент) | Частично: есть точки загрузки/выгрузки | Расширить модель: типы точек (АЗС, граница, отдых, питание), нормативы на каждую |
| 6 | **Чек-ин на точках** (авто по GPS + ручной) | Частично: GPS-отметки прибытия/убытия есть в мобилке | Доработать: автоматический чек-ин по геозоне + ручное подтверждение, привязка к типу операции |
| 7 | **Фиксация операций** (погрузка, слив, граница, заправка, остановка) | Частично: старт/прибытие/убытие есть | Расширить: добавить конкретные типы операций с замером длительности |
| 8 | **Причины задержки** из справочника | Нет | **Новый модуль:** справочник причин (очередь, ремонт, погода, граница, поломка и т.д.), выбор в мобилке и на вебе |
| 9 | **Рабочее место диспетчера** (карта + статусы + алерты + инструменты) | Частично: мониторинг с картой есть | Доработать: алерты по отклонениям, инструменты реакции (комментарий водителю, задача, фиксация причины), фильтры по "красным" маршрутам |
| 10 | **Мобилка водителя** (назначение, навигация, чек-ин, фото-документы) | Частично: основные функции есть | Доработать: навигация по маршруту с контрольными точками, отображение ETA и отставания, загрузка ТТН/CMR/акт слива |
| 11 | **Готовность к следующей погрузке** | Нет | **Новый модуль:** расчет "когда машина будет на заводе и готова к погрузке" |
| 12 | **Отчеты по нормативам** (% в нормативе, среднее отклонение, рейтинг проблемных точек) | Нет в таком виде | **Новые отчеты:** нормативная аналитика по этапам, рейтинг задержек |
| 13 | **Оборачиваемость цикла** (среднее время рейса, время до след. погрузки) | Нет | **Новый отчет** |
| 14 | **Простой по причинам** | Нет | **Новый отчет** с привязкой к справочнику причин |
| 15 | **Пробег полезный vs холостой, км/тонна** | Нет | **Новый отчет** с расчетами по GPS-данным |
| 16 | **Эффективность водителей** (нарушения, красные участки, скорость) | Частично: есть сравнение водителей | Доработать: привязать к нормативам, добавить "красные участки" по водителям |
| 17 | **Экспортная аналитика** (время прохождения границы, перцентили) | Нет | **Новый отчет** с детализацией по пунктам пропуска |
| 18 | **Финансово-управленческие отчеты** (себестоимость рейса, стоимость простоя, прогноз загрузки) | Частично: финансовые отчеты есть | Доработать: расчет себестоимости, прогнозирование загрузки парка |
| 19 | **Модуль ТО** (план-график, масло/шины, интеграция с рейсами) | Нет | **Новый модуль** (следующий этап) |

## 3. ГРУППИРОВКА ДОРАБОТОК ПО БЛОКАМ

### БЛОК А: Модель рейса и нормативы (ядро)
- Типы рейсов (экспорт / внутри РК)
- Цикличность: рейс → возврат → готовность к следующему
- Контрольные точки с типами (АЗС, граница, отдых, клиент)
- Справочник нормативов (время на погрузку, слив, границу, участок)
- Автоматический расчет ETA/ETD по нормативам

### БЛОК Б: Контроль и автоперерасчет
- Сравнение факт/план на каждой контрольной точке
- Подсветка "красных участков" (бэк + фронт + мобилка)
- Автоперерасчет оставшегося графика при отклонении
- Расчет готовности к следующей погрузке

### БЛОК В: Рабочее место диспетчера
- Алерты по отклонениям (push + UI)
- Инструменты реакции: комментарий, задача, причина задержки
- Фильтрация по "проблемным" рейсам
- Сводная панель статусов всех машин

### БЛОК Г: Мобильное приложение водителя
- Чек-ин на контрольных точках (GPS-геозона + ручной)
- Фиксация типа операции (погрузка/слив/граница/АЗС/отдых)
- Выбор причины задержки из справочника
- Отображение ETA и текущего отставания
- Загрузка фото-документов (ТТН, CMR, акт слива)

### БЛОК Д: Аналитика и отчеты
- Нормативная аналитика (% в нормативе, среднее отклонение)
- Рейтинг проблемных точек
- Оборачиваемость цикла
- Простой по причинам
- Полезный пробег vs холостой, км/тонна
- Эффективность водителей с привязкой к нормативам
- Экспортная аналитика (границы, перцентили)
- Себестоимость рейса (план/факт)
- Прогноз загрузки парка

### БЛОК Е: Модуль ТО (следующий этап)
- План-график ТО по моточасам/пробегу/календарю
- Учет замен (масло, шины, расходники)
- Блокировка назначения машины при просроченном ТО
- Заявки на ремонт от водителя + фото
- Отчеты по поломкам, стоимости ремонта

## 5. ИНТЕГРАЦИЯ ИИ

### Где ИИ дает максимальную пользу для MONOTRANS

| Задача | Что делает ИИ | Приоритет |
|--------|--------------|-----------|
| **Прогноз ETA** | ML-модель предсказывает реальное время прибытия на основе исторических данных, погоды, дня недели, загрузки дорог | Высокий |
| **Обнаружение аномалий** | Автоматическое выявление необычных задержек, отклонений от маршрута, подозрительных остановок | Высокий |
| **Оптимизация маршрутов** | Подбор оптимального маршрута с учетом контрольных точек, пробок, границ | Средний |
| **Прогноз загрузки парка** | Предсказание доступности машин, оптимальное планирование рейсов | Средний |
| **Рекомендации диспетчеру** | ИИ-ассистент подсказывает корректирующие действия при отклонениях | Средний |
| **Анализ причин задержек** | Кластеризация и выявление системных проблем (конкретная граница, смена на заводе, время суток) | Средний |
| **Предиктивное ТО** | Прогноз поломок по данным телематики и истории ремонтов | Низкий (след. этап) |

### Рекомендуемый подход к ИИ

**Фаза 1 — Сбор данных (параллельно с Этапами 1-2):**
- Накопление данных по рейсам, нормативам, факту, задержкам, координатам
- Структурирование данных для аналитики

**Фаза 2 — Базовые модели (Этап 3):**
- ML-прогноз ETA на основе исторических данных
- Детекция аномалий (отклонения от нормы)
- Рекомендуемые ИИ-платформы: Python + scikit-learn / TensorFlow для моделей, отдельный микросервис

**Фаза 3 — Продвинутая аналитика (после запуска):**
- LLM-ассистент для диспетчера (анализ ситуации и рекомендации на естественном языке)
- Оптимизация распределения рейсов по автопарку
- Предиктивное ТО

### Рекомендуемые ИИ-решения

| Задача | Технология | Примечание |
|--------|-----------|------------|
| Прогноз ETA | Python + XGBoost / LightGBM | Лучшие результаты для табличных данных |
| Обнаружение аномалий | Isolation Forest / DBSCAN | Хорошо работает с GPS и временными рядами |
| Оптимизация маршрутов | OR-Tools (Google) / OSRM | Проверенные решения для логистики |
| ИИ-ассистент диспетчера | Claude API / GPT API | Анализ ситуации и рекомендации на русском языке |
| Предиктивное ТО | TensorFlow + временные ряды | После накопления данных по ТО |

### Сроки по ИИ

| Фаза | Срок | Ресурсы |
|------|------|---------|
| Фаза 1: сбор данных | Параллельно (0 доп. недель) | Архитектор данных |
| Фаза 2: базовые ML-модели | 6–8 недель | 1 ML-инженер |
| Фаза 3: продвинутый ИИ | 8–12 недель | 1 ML-инженер + 1 бэкенд |
